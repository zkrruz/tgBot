# Telegram ATS Bot для кандидатов

Бот помогает кандидату понять, насколько его резюме релевантно рынку системных аналитиков и как оно пройдет ATS-фильтры. Пользователь загружает резюме, выбирает краткий или полный отчет и получает PDF с оценкой и планом улучшений.

## Пользовательский сценарий

1. Кандидат отправляет резюме файлом: PDF, DOC, DOCX или RTF.
2. Бот извлекает текст резюме.
3. Кандидат выбирает тип отчета: краткий или полный.
4. Бот сравнивает резюме с рыночным датасетом вакансий системного аналитика.
5. Кандидат получает PDF-отчет с баллом, уровнем, рисками и рекомендациями.

## Что оценивает бот

- общий ATS-балл и грейд;
- сильное, среднее и низкое соответствие вакансиям из датасета;
- предполагаемый уровень: Junior, Middle или Senior;
- доменный фокус;
- покрытие ключевых и технических навыков;
- структуру резюме;
- первое впечатление рекрутера;
- недостающие требования рынка;
- быстрый и глубокий план улучшений.

## Рыночный датасет

Датасет хранится в SQLite и пополняется вакансиями. Если база пустая, бот использует встроенный базовый профиль системного аналитика, но для реального продукта лучше добавить 100+ вакансий.

Команды администратора:

- `/add_market_vacancy` - добавить вакансию в датасет;
- `/dataset` - посмотреть размер датасета;
- `/vacancies` - показать список вакансий;
- `/delete_vacancy ID` - удалить вакансию.

Если `ADMIN_IDS` пустой, админские команды доступны всем. Для продакшена обязательно укажите Telegram ID администраторов.

## Установка

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
copy .env.example .env
```

Заполните `.env`:

```env
BOT_TOKEN=токен_от_BotFather
OPENAI_API_KEY=опционально
OPENAI_MODEL=gpt-4.1-mini
ADMIN_IDS=123456789
DATABASE_PATH=data/ats_bot.sqlite3
REPORTS_DIR=reports
```

`OPENAI_API_KEY` можно не указывать. Тогда бот использует локальную keyword-based оценку. С OpenAI отчет становится более содержательным и похожим на экспертный разбор.

## Запуск

```bash
python main.py
```

## Примечания по DOC

Старый `.doc` является бинарным форматом. Для его чтения на сервере нужен один из инструментов: `antiword`, `catdoc` или LibreOffice (`soffice`). Форматы PDF, DOCX и RTF читаются Python-библиотеками из `requirements.txt`.
